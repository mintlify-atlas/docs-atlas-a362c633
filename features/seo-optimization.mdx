## Overview

Polaris Clima implements comprehensive SEO optimization through structured data (Schema.org), dynamic metadata generation, sitemap configuration, and geo-targeting meta tags.

## Structured Data Schemas

Two JSON-LD schemas are embedded on every page to help search engines understand the business:

1. **LocalBusinessSchema** (HVACBusiness)
2. **ServiceSchema** (Service catalog)

### LocalBusinessSchema

**Location**: `components/LocalBusinessSchema.tsx`

**Schema Type**: `HVACBusiness` (specialized LocalBusiness type)

### Geo-Targeting Meta Tags

<CodeGroup>
```tsx Geo Meta Tags
<head>
  {/* Geographic location */}
  <meta name="geo.region" content="CL-RM" />
  <meta name="geo.placename" content="Santiago" />
  <meta name="geo.position" content="-33.4489;-70.6693" />
  <meta name="ICBM" content="-33.4489, -70.6693" />
  
  {/* Browser theme color */}
  <meta name="theme-color" content="#4f46e5" />
  
  {/* Disable automatic phone number detection */}
  <meta name="format-detection" content="telephone=no" />
</head>
```
</CodeGroup>

<Info>
**CL-RM** = Chile - Región Metropolitana (ISO 3166-2 code)

Geo tags help search engines and directories understand the business's physical location and service area.
</Info>

```xml Sitemap Structure

  </url>

  <!-- Commune Pages (Priority: 0.8) -->
  <url>
    <loc>https://www.polarisclima.cl/aire-acondicionado/santiago</loc>
    <lastmod>2026-02-26</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
  </url>
  
  <!-- ... 17 more commune URLs -->
  
</urlset>
```
</CodeGroup>

### Priority Hierarchy

| Page Type | Priority | Rationale |
|-----------|----------|----------|
| Homepage | 1.0 | Most important page |
| Service Pages | 0.9 | Core offerings |
| Commune Pages | 0.8 | Local landing pages |
| General Pages | 0.5-0.7 | Supporting content |

### Change Frequency

All pages set to `monthly` as content is relatively stable (service descriptions, contact info, etc.).
