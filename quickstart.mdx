## Get Started in 5 Minutes

This guide will walk you through setting up Polaris Clima on your local machine, from cloning the repository to making your first changes.

<Note>
  Make sure you have Node.js 20+ and pnpm installed on your system before proceeding.
</Note>

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 20+** - [Download Node.js](https://nodejs.org/)
- **pnpm** - Fast, disk space efficient package manager
- **MySQL 8.0+** - For contact form data storage
- **Git** - Version control

<Steps>

### Clone the Repository

First, clone the Polaris Clima repository to your local machine:

```bash
git clone <repository-url>
cd polaris-clima
```

Alternatively, if you're starting from the workspace:

```bash
cd ~/workspace/source
```

### Install Dependencies

Polaris Clima uses pnpm as its package manager. Install all dependencies:

```bash
pnpm install
```

This will install:
- Next.js 16.1.1 and React 19
- Tailwind CSS 4 and PostCSS
- Framer Motion for animations
- MySQL2 for database connectivity
- TypeScript and development tools

<Warning>
  The project uses pnpm workspaces. Make sure you're running `pnpm install` (not `npm install`) to avoid dependency conflicts.
</Warning>

### Configure Environment Variables

Create a `.env.local` file in the root directory with your database credentials:

```bash title=".env.local"
# MySQL Database Configuration
DB_HOST=localhost
DB_USER=your_mysql_user
DB_PASSWORD=your_mysql_password
DB_NAME=polaris_clima
```

<Note>
  These environment variables are only used during development when running API routes. The static export won't include API functionality unless deployed with server-side capabilities.
</Note>

### Set Up the Database

Create the MySQL database and table for contact form submissions:

```sql
-- Create the database
CREATE DATABASE polaris_clima CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE polaris_clima;

-- Create the contactos table
CREATE TABLE contactos (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nombre VARCHAR(100) NOT NULL,
  apellido VARCHAR(100) NOT NULL,
  correo VARCHAR(150) NOT NULL,
  telefono VARCHAR(20) NOT NULL,
  mensaje TEXT,
  origen VARCHAR(50),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_created_at (created_at),
  INDEX idx_origen (origen)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### Start the Development Server

Launch the Next.js development server:

```bash
pnpm dev
```

The server will start on `http://localhost:3000`. You should see output like:

```
  ▲ Next.js 16.1.1
  - Local:        http://localhost:3000
  - Environments: .env.local

 ✓ Ready in 1.2s
```

### Verify the Installation

Open your browser and navigate to:

- **Homepage**: http://localhost:3000
- **Contact Page**: http://localhost:3000/contacto
- **Location Page**: http://localhost:3000/aire-acondicionado/santiago
- **Service Page**: http://localhost:3000/servicios/instalacion

You should see the Polaris Clima homepage with:
- Responsive navigation header
- Hero section with animations
- Brand carousel
- Service cards
- Contact form
- Testimonials section

</Steps>

Let's customize the main hero section. Open `app/page.tsx`:

```typescript title="app/page.tsx"

```

The homepage is composed of reusable components. Each section can be edited independently in the `components/` directory.

### Customize the Theme Colors

Edit the Tailwind configuration to match your brand:

```typescript title="tailwind.config.ts"
const config: Config = {
  theme: {
    extend: {
      colors: {
        primary: '#137fec',           // Primary brand color
        'background-light': '#f6f7f8', // Light background
        'background-dark': '#101922',  // Dark background
        'text-primary': '#111418',     // Primary text
        'text-secondary': '#0ECD61',   // Secondary accent
      },
    },
  },
};
```

Change these colors to match your brand identity. The changes will apply across the entire application.

### Add a New Service Page

Create a new service by adding a page in `app/servicios/`:

```bash

touch app/servicios/nuevo-servicio/page.tsx
```

```typescript title="app/servicios/nuevo-servicio/page.tsx"
import { Metadata } from 'next';
```

### Test the Contact Form

Navigate to http://localhost:3000/contacto and submit a test form:

1. Fill in all required fields (nombre, apellido, email, teléfono)
2. Add an optional message
3. Click "ENVIAR"

The form submission flow:
```typescript title="app/api/contacto/route.ts" {37-51}
export async function POST(req: Request) {
  try {
    // Rate limiting by IP
    const ip = req.headers.get('x-forwarded-for')?.split(',')[0]?.trim()
      || req.headers.get('x-real-ip')
      || 'unknown';

    if (isRateLimited(ip)) {
      return NextResponse.json(
        { error: 'Demasiados envíos. Intenta de nuevo en unos minutos.' },
        { status: 429 }
      );
    }

    const body = await req.json();
    const { nombre, apellido, email, telefono, mensaje, origen } = body;

    // Validation...
    
    connection = await getConnection();
    await connection.execute(
      `INSERT INTO contactos (nombre, apellido, correo, telefono, mensaje, origen)
       VALUES (?, ?, ?, ?, ?, ?)`,
      [nombre.trim(), apellido.trim(), email.trim().toLowerCase(), 
       telefono.trim(), mensajeLimpio, origen || 'desconocido']
    );

    return NextResponse.json({ success: true });
  } catch (error) {
    console.error('Error DB:', error);
    return NextResponse.json({ error: 'Error al guardar' }, { status: 500 });
  }
}
```

The form includes:
- Email and phone validation
- Rate limiting (4 requests per 10 minutes per IP)
- XSS protection and input sanitization
- MySQL database storage

Generate the static export:


## Debugging Tips

### Check Database Connection

If contact forms aren't working:

```bash
# Test MySQL connection
mysql -u your_mysql_user -p polaris_clima

# Check if table exists
SHOW TABLES;

# View recent submissions
SELECT * FROM contactos ORDER BY created_at DESC LIMIT 10;
```

### View Console Logs

API route logs appear in the terminal where you ran `pnpm dev`:

```bash
POST /api/contacto 200 in 142ms
```

Client-side logs appear in the browser console (F12).

### Clear Next.js Cache

If you encounter strange build issues:

```bash
rm -rf .next
pnpm dev
```

## Troubleshooting

### Port Already in Use

```bash
# Use a different port
pnpm dev -- -p 3001
```

### Module Not Found Errors

```bash
# Clear node_modules and reinstall
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### TypeScript Errors

```bash
# Check TypeScript configuration
pnpm tsc --noEmit
```

### Database Connection Failed

Verify your `.env.local` credentials and ensure MySQL is running:

```bash
# Check MySQL status (macOS)
brew services list | grep mysql

# Check MySQL status (Linux)
sudo systemctl status mysql
```
