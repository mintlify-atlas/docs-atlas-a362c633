## Overview

The `ContactSection` component provides a comprehensive contact form with real-time validation, error handling, and success feedback. It integrates with a custom `useContactForm` hook for state management and includes a decorative image section.

<Warning>
  This is a **client component** (`'use client'`) and cannot be used in server components or during SSR.
</Warning>

## Component Features

- Real-time field validation with error messages
- Loading states during form submission
- Success/error feedback alerts
- Two-column responsive layout (form + content)
- Integration with backend API
- Source tracking (page origin)

## Usage

```tsx
import ContactSection from '@/components/ContactUs';

export default function ContactPage() {
  return (
    <main>
      <ContactSection />
    </main>
  );
}
```

<Note>
  The component is self-contained with no required props. Form behavior is configured via the `useContactForm` hook.
</Note>

## useContactForm Hook

The component uses a custom hook for form state management:

### Hook Signature

```tsx
import { useContactForm } from '@/hooks/useContactForm';

const {
  loading,        // boolean: Form submission in progress
  success,        // boolean: Form submitted successfully
  errorMessage,   // string: Error message to display
  handleSubmit,   // (e: FormEvent) => Promise<void>
  handleChange,   // (e: ChangeEvent) => void
  fieldProps      // (name: string) => FieldProps
} = useContactForm('pagina-principal'); // origen param
```

### Hook Parameters

<ParamField path="origen" type="string" required>
  Source page identifier for tracking form submissions.
  
  **Examples:**
  - `'pagina-principal'` - Homepage
  - `'servicios'` - Services page
  - `'contacto'` - Dedicated contact page
  
  This value is sent to the API for analytics.
</ParamField>

### Hook Return Values

<Tabs>
  <Tab title="State">
    ```tsx
    loading: boolean      // True during API call
    success: boolean      // True after successful submission
    errorMessage: string  // Error message (empty if no error)
    ```
  </Tab>
  
  <Tab title="Handlers">
    ```tsx
    handleSubmit: (e: FormEvent) => Promise<void>
    // Validates form, calls API, handles response
    
    handleChange: (e: ChangeEvent) => void
    ```

## Form Validation

Validation occurs in two scenarios:

<Steps>
  <Step title="On Blur">
    When a user leaves a field (onBlur event), the field is validated immediately.
  </Step>
  
  <Step title="On Submit">
    All fields are validated before API call. Submission blocked if any errors exist.
  </Step>
</Steps>

### Validation Rules

```tsx
function validateField(name: string, value: string): string {
  switch (name) {
    case 'nombre':
    case 'apellido':
      if (!value.trim()) return `El ${name} es obligatorio`;
      if (!/^[a-zA-ZáéíóúÁÉÍÓÚñÑ\s]+$/.test(value))
        return `El ${name} solo puede contener letras`;
      return '';
    
    case 'email':
      if (!value.trim()) return 'El correo es obligatorio';
      if (!/\S+@\S+\.\S+/.test(value)) return 'El correo no es válido';
      return '';
    
    case 'telefono':
      if (!value.trim()) return 'El teléfono es obligatorio';
      if (!/^\+?[0-9\s()-]{7,15}$/.test(value)) return 'El teléfono no es válido';
      return '';
    
    default:
      return '';
  }
}
```

## API Integration

### Endpoint Configuration

```tsx

```

<ParamField path="NEXT_PUBLIC_API_CONTACTO" type="string" default="/api/contacto">

  ```bash
  NEXT_PUBLIC_API_CONTACTO=https://api.polarisclima.cl/contacto
  ```

### Request Format

```tsx
POST /api/contacto
Content-Type: application/json

  "nombre": "Juan",
  "apellido": "Pérez",
  "email": "juan@example.com",
  "telefono": "+56 9 1234 5678",
  "mensaje": "Consulta sobre instalación",
  "origen": "pagina-principal"
}
```

### Response Handling

<CodeGroup>
  ```json Success (200)
  {
    "success": true
  }
  ```
  
  ```json Rate Limited (429)
  {
  ```

  ```json Server Error (500)

  }
  ```

## Alert Components

The form uses two alert components:

### Success Alert

```tsx
{success && (
  <FormAlert 
    type="success" 
    message="¡Gracias por contactarnos! Te responderemos pronto." 
  />
)}
```

- Green background with checkmark icon
- Auto-dismisses after 5 seconds
- Form fields are cleared

### Error Alert

```tsx
{errorMessage && (
  <FormAlert 
    type="error" 
    message={errorMessage} 
  />
)}
```

- Red background with X icon
- Remains until user interacts with form
- Specific error messages based on failure type

### Add Additional Fields

To add a new field:

1. Update `FormData` type in hook:
   ```tsx
   type FormData = {
     nombre: string;
     apellido: string;
     email: string;
     telefono: string;
     mensaje: string;
     newField: string; // Add here
   };
   ```

2. Add validation in `validateField()`

3. Add field to form:
   ```tsx
   <InputField placeholder="New Field*" {...fieldProps('newField')} />
   ```
